# 产品名称

Water Me

# Mission

帮助人们建立健康的工作习惯。

# Vision

一个真正理解你工作状态的伙伴——不是在你忙碌时打扰你，而是在你最需要的时候轻轻提醒你照顾自己。

# Slogan

**Humans need watering, too.**

> 人类，也需要浇水。

# Philosophy

人们每天都会照顾自己的工作、学习、家人，却常常忘记照顾自己。

Water Me 希望通过最自然的方式，帮助用户建立健康的工作习惯。

它不是一个计时器。

也不是一个喝水软件。

它是一位默默照顾你的伙伴。

## 为什么提醒会失效？

人不是因为忘记喝水，而是主动选择忽略提醒。

大脑会瞬间决策："我知道了，等会儿。"然后 30 分钟过去了。1 小时过去了。最后一天都没喝。

所以问题从来不是：如何提醒？

而是：**如何让用户愿意响应提醒？**

## 从「提醒软件」到「习惯软件」

- **提醒软件**关注的是「通知有没有弹出来」。
- **习惯软件**关注的是「用户有没有真正完成这个行为」。

后者才更有长期价值，也更符合桌宠、成长系统和游戏化设计的发展方向。

## 行为设计

不是要求"喝水"，而是邀请用户"去接一杯水，活动一下身体"。

提醒内容应该是：

> 去给自己接一杯水。

而不是：

> 请喝水。

这是两个完全不同的心理感受。

只要站起来、去接水、走几十步、眼睛离开屏幕——三个目标全部完成。

## 拟人化提醒

产品理念是 **Humans need watering, too.**，提醒可以反过来：

```
🌱 I think you forgot me.
🌱 Could you water me?
🌱 I'll wait here.
```

真正需要浇水的是用户自己，但说的是植物。

这种拟人化让提醒没有命令感。

# Design Principles

## ① Respect the User

尊重用户。

永远允许 Skip。永远不强迫。

## ② Context First

所有提醒：基于真实工作状态。不是时钟。

## ③ Less is More

减少通知。增加有效提醒。

## ④ Health over Productivity

Water Me 不是提高工作效率。而是保持健康。

## ⑤ Build Habits, Not Pressure

帮助形成习惯。不是制造压力。

以后任何功能违反 Principle，直接不做。

例如：排行榜。违反 Respect。直接 Out。

# Product Personality

Water Me 到底是什么？

医生？×

教练？×

领导？×

妈妈？×

应该是：**朋友。** 或者：**默默陪伴你的植物。**

人格：

```
Warm
Gentle
Patient
Never blame
```

永远不会说：你今天失败了。

只会说：

```
🥺 I'll wait here.
```

这种人格以后所有文案全部统一。

递进情绪：

```
🌱 Water Me.
🌱 Still waiting...
🥺 I'm getting thirsty.
😭 Please...
💀 ...
```

不是越来越烦，而是越来越有情绪。

这就是桌宠以后存在的意义。

# Long-term Direction

Water Me 的终极方向不是一个提醒工具，而是一个健康习惯系统。

```
Activity Engine（核心引擎）
↓
Health Activity（统一行为模型）
↓
表现层（Overlay / 桌宠 / 通知 / 移动端）
```

所有健康行为统一为 Health Activity：

```
Health Activity
├── Get Water
├── Stand Up
├── Eye Break
├── Stretch
├── Walk
├── ...
```

任何 Health Activity 都是插件。引擎不变，行为可扩展。

表现层可以替换：

```
Overlay
↓
Desktop Pet
↓
System Notification
↓
Mobile Push
```

业务逻辑完全不用动。
